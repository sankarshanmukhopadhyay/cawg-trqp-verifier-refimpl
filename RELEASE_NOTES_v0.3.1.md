# Release Notes: v0.3.1

This release focuses on packaging quality and public-repo usability.

## Highlights

- tightened the README to make the repository easier to understand on first read
- refreshed repository tree documentation
- added a clearer release-readiness assessment
- preserved the executable reference implementation structure from v0.3.0

## Why this release matters

The repository now presents a cleaner public-facing story:

1. **What the repo is for**  
   To show, in executable form, how TRQP sits between identity extraction and trust outcome synthesis in a CAWG/C2PA workflow.

2. **What the repo already does**  
   Demonstrates standard, edge, and high-assurance verification profiles using a mock TRQP service, cache layer, and snapshot loader.

3. **What remains open**  
   Real CAWG/C2PA parsing, signed snapshots, HTTP transport, stronger revocation handling, and richer conformance coverage.

## Release posture

This is a strong GitHub-ready reference skeleton for architectural and implementation discussions.
