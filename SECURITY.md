# Security Policy

## Supported version

Security fixes are applied to the current main branch and the latest published version. Older versions may be unsupported unless a maintainer states otherwise.

## Reporting a vulnerability

Do not open a public issue for an undisclosed vulnerability. Use GitHub private vulnerability reporting when enabled, or contact the repository owner through the profile contact channel. Include affected versions, reproduction steps, impact, and any proposed mitigation.

## Response and evidence

Validated reports will be triaged for severity, affected authority surfaces, evidence integrity, compatibility impact, and required revocation or supersession actions. Public disclosure should follow remediation and coordinated notification where appropriate.
